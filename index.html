Aspectos fuera del alcance

            
    
              
  • No crea puntos de acceso autorizados.
    • 
              
  • No consulta catálogos ni fuentes externos.
    • 
              
  • No asigna vocabulario controlado normalizado.
    • 
            
            

            
El profesional mantiene el control. Esta herramienta genera un registro preliminar estructurado que requiere revisión, validación y enriquecimiento por parte de un catalogador antes de su incorporación al catálogo.


            

              
Privacidad y datos

              
La aplicación envía el texto e imágenes al servidor configurado y el servidor consulta la API de OpenAI para extraer y estructurar metadatos. No coloque claves de API en el navegador ni en GitHub Pages; configúrelas solo como variables de entorno del backend.

            


            
Recomendaciones para mejorar la calidad del registro

            
    
              
  • Verifique siempre los campos 1xx/7xx: los nombres de autor pueden requerir normalización según las reglas de catalogación de su institución.
    • 
              
  • Revise las materias (650): el sistema sugiere materias basadas en el texto, pero no aplica vocabularios controlados (LCSH, EMBUS, etc.).
    • 
              
  • Complete los puntos de acceso secundarios: el sistema no genera encabezamientos adicionales (700/710) más allá de autores secundarios explícitos.
    • 
              
  • Valide las clasificaciones 050/082: las sugerencias de LC y Dewey son orientativas. Consulte fuentes autorizadas (ClassWeb, WebDewey) para confirmar.
    • 
              
  • Revise el campo 264: la secuencia de publicación (lugar, editor, fecha) puede requerir ajustes según RDA o las políticas locales.
    • 
              
  • Complete el campo 300: descripción física (ilustraciones, dimensiones, material acompañante) no siempre se extrae automáticamente.
    • 
              
  • Use la vista de edición para hacer ajustes directos al registro MARC antes de descargarlo.
    • 
            

          

        


      

    


    

      
      

      

        
 ¿Cómo funciona?

        
1
Sube un PDF (se renderizan las primeras páginas como imágenes) o imágenes organizadas (portada, página legal, última página).

        
2
OCR Visual: un modelo OpenAI con visión analiza las imágenes y extrae el texto visible.

        
3
Estructuración: un modelo OpenAI interpreta el texto y lo estructura en metadatos MARC21 según reglas RDA.

        
4
El sistema genera un registro MARC21 válido con campos, indicadores y subcampos.

        
5
Revisa, edita y descarga el registro en formato MARCXML para importar en Koha u otros ILS.

      


      

        
 Pipeline de dos agentes

        
Este sistema utiliza un pipeline de dos agentes de IA:

        
    
          
  • Agente OCR (OPENAI_OCR_MODEL): modelo con visión especializado en extraer texto de imágenes de documentos.
    • 
          
  • Agente Estructurador (OPENAI_STRUCTURING_MODEL): modelo que aplica reglas de catalogación RDA/ISBD para estructurar metadatos en MARC21.
    • 
        

        
Nuevo

        
El PDF se convierte automáticamente a imágenes para el análisis visual. Cada etapa tiene reintentos automáticos si la respuesta no es válida.
